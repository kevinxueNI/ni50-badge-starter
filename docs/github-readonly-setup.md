# GitHub Read-Only Sharing

## Objective

Use this repository as a private GitHub code share package and grant selected people read-only access.

## Recommended Setup

1. Keep the repository `Private`.
2. Only invite the people who need access.
3. Grant them `Read` permission only.

## GitHub Steps

1. Open the repository on GitHub.
2. Go to `Settings`.
3. Open `Collaborators and teams`.
4. Click `Add people` or `Add teams`.
5. Select the user or team.
6. Set permission to `Read`.
7. Send the invitation.

## What Read Access Allows

People with `Read` access can:

- view the code,
- clone or download it,
- read the documentation.

They cannot:

- push changes,
- merge pull requests,
- modify repository settings,
- delete branches.

## Recommended Maintenance Notes

1. Keep the default branch as `main`.
2. Do not grant `Write` unless collaboration is actually required.
3. Keep README and quick start current so the repository is understandable without external context.
